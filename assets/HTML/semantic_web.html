RDF Serialization

At this stage, the previously generated URIs are used to construct statements in RDF (Resource Description Framework), specifically utilizing the Turtle serialization format. RDF is a foundational standard of the Semantic Web designed to represent information about resources in a structured, machine-readable format. It expresses data in the form of triples—subject, predicate, and object—that enable the linking of disparate data sources through shared identifiers.

The generated RDF triples incorporate not only the internal URIs associated with the selected items but also reference external authorities and controlled vocabularies, establishing meaningful links between local data and broader knowledge networks. This structured approach facilitates data interoperability, semantic enrichment, and integration across platforms, ensuring that the described items are situated within a larger web of contextual relationships


URI Collection

Specific items from the curated list are selected to serve as RDF subjects. For each of these items and their related entities, unique URIs (Uniform Resource Identifiers) are systematically constructed. These URIs act as persistent and unambiguous references within the RDF framework, enabling detailed descriptions of properties and interrelationships according to semantic web principles.

Through the assignment of URIs, each resource becomes part of a globally addressable information space. This ensures that data can be linked, queried, and reused across different platforms and knowledge domains, promoting interoperability and enriching the context in which each item exists.
